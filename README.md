## 프로젝트 소개

### 📌 프로젝트 배경
- **현대인의 VDT 증후군 증가**: 장시간 컴퓨터와 스마트폰을 사용하는 대학생 및 직장인들이 늘어남에 따라, 고개가 앞으로 숙여지는 '거북목 증후군(Turtle Neck Syndrome)' 환자가 급증하고 있습니다.
- **자세 인지의 한계**: 거북목은 통증이 심해지기 전까지 스스로 인지하기 어렵고, 의식적으로 자세를 고치더라도 시간이 지나면 다시 나쁜 자세로 돌아가는 무의식적 습관의 문제가 있습니다.
- **실실간 피드백 장치의 필요성**: 기존의 자세 교정 밴드나 웨어러블 장비는 상시 착용해야 하는 불편함이 있으므로, 일상적인 웹캠 환경에서 사용자를 방해하지 않고 직관적인 시청각 경고를 주는 시스템이 필요합니다.

## 시스템 구조

```
[웹캠 (사용자 촬영)]
            │
            ▼ (실시간 영상 프레임)
     [맥북 / PC 환경] 
      ├─ OpenCV (영상 전처리 & RGB 변환)
      ├─ PyTorch 기반 MobileNetV2 
      │    └─ 5초 간격 이미지 추론 (거북목 vs 바른자세 이진 분류)
      ├─ 데이터 수집 피드백 루프 (키보드 'c' 또는 'i' 입력)
      │    └─ 맞춤/틀린 데이터 자동 분류 저장 (`collected_data/`)
      │
      └─ pyfirmata2 시리얼 통신 (USB, Unix Port)
            │
            ▼ (제어 명령어 전송)
       [아두이노 Uno]
            ├─ 11번 핀 (PWM): 수동 부저 ──▶ [거북목 감지 시 경고음 가동]
            └─ 10번 핀 (Digital): LED ──▶ [바른 자세 유지 시 초록불 점등]
```
> 위 다이어그램을 실제 시스템에 맞게 수정하세요. 필요하다면 이미지 파일로 대체할 수 있습니다.

## 구현 방법

### 1. Vision AI 모델

- **모델 선택 이유**: PC와 웹캠 환경에서 실시간 스트리밍 영상(FPS)을 끊김 없이 처리하면서도, 높은 정확도를 확보하기 위해 구글의 대표적인 경량화 모델인 MobileNetV2를 채택했습니다.
- **전이학습***: 이미 거대한 데이터셋(ImageNet)으로 학습된 사전 가중치(MobileNet_V2_Weights.DEFAULT)를 가져와 특성 추출기(Feature Extractor)의 가중치를 고정(requires_grad = False)하고, 최종 분류기(Classifier) 레이어만 우리 프로젝트의 클래스 수(2개: 거북목, 바른자세)에 맞게 nn.Linear로 교체하여 소량의 데이터로도 빠르게 고성능 모델을 구축했습니다.
- **데이터셋 및 전처리**: 측면 앉은 자세 이미지를 인터넷에서 수집하여 저장후 활용했습니다. 입력 이미지는 모델 규격에 맞춰 224x224 크기로 리사이즈하고, 토치 텐서 변환 후 ImageNet 평균값과 표준편차를 기준으로 정규화(Normalize)를 수행했습니다. 오버피팅을 방지하고 정확한 학습 손실 추이를 보기 위해 전 과정 데이터로 15 에포크(Epoch) 동안 훈련을 진행했습니다.


### 2. 아두이노 제어 로직

- 수신된 명령어에 따라 부저가 울리거나 LED 작동
- 거북목으로 판단되면 부저가 울림
- 바른자세로 판단되면 LED가 켜짐
- (회로도 이미지가 있다면 삽입)

## 향후 개선 사항

- 다양한 오판단 케이스 학습: 수집 기능(i 키)을 통해 축적된 오답 데이터를 기반으로 가중치를 고도화하여 의자 각도, 어두운 조명 환경에서의 인식 성능 강화
- 웹 대시보드 및 통계화: 실시간으로 축적되는 거북목 빈도 데이터를 차트로 시각화하여 사용자가 하루 동안 얼마나 바른 자세를 유지했는지 모니터링할 수 있는 UI 구축
- 알림 커스텀화: 지속적인 거북목 감지 시 부저 음량을 단계별로 키우거나, 맥북 상단 알림창(Notification) 팝업 메시지와 연동 기능 추가
